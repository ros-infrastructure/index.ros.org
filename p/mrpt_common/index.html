Changelog for package mrpt_common

3.0.4 (2026-06-17)

3.0.3 (2026-06-15)

3.0.2 (2026-06-11)

  • fix: BUILD_TESTING=OFF default was ignored due to include(CTest) ordering CTest's own option(BUILD_TESTING ... ON) ran first and created the cache variable, making our subsequent option(BUILD_TESTING ... OFF) a no-op. This left BUILD_TESTING=ON on the ROS build farm (which doesn't pass -DBUILD_TESTING=OFF), causing find_package(GTest REQUIRED) to fail and break the mrpt_core binary build. Also fix CI workflows to pass the correct -DBUILD_TESTING=ON flag (macOS/Windows previously used the non-existent MRPT_BUILD_TESTING, and Linux relied on the buggy default).
  • Contributors: Jose Luis Blanco-Claraco
